# AI 코딩 에이전트 비교

AI 코딩 에이전트 간의 트레이드오프를 분석하는 프레임워크. 속도, 코드 품질, 자율성, 지시 준수 등 여러 축에서 에이전트를 비교한다.

## 핵심 트레이드오프 축

### 속도 vs 품질
- [[claude-code]]: 빠르지만 "핵 패치"와 헬퍼 함수 남발 경향
- [[codex]]: 3~4배 느리지만 더 깨끗하고 체계적인 코드 생성
- 세션당 생산성은 Claude가 높지만, 누적 기술 부채는 Codex가 낮음

### 자율성 vs 인터랙티브
- Claude: 감독(babysitting) 필요, 출력을 실시간으로 모니터링해야 함
- Codex: "실행시키고 결과만 리뷰"가 가능한 수준의 자율성

### 지시 준수
- Claude: CLAUDE.md를 세션당 최소 1회 이상 무시하는 경향
- Codex: AGENTS.md를 한 번도 무시하지 않으며, 세션 중 오버라이드도 거부

### 작업 완결성
- Claude: 반빈번하게 작업을 중간에 방치 (예: 8개 중 일부만 마이그레이션)
- Codex: 높은 작업 완결성

## 공통 전제

두 도구 모두 **SWE 역량이 없으면 좋은 결과를 내지 못한다**. AI 코딩 에이전트는 숙련된 엔지니어의 생산성을 증폭하는 도구이지, 역량을 대체하는 도구가 아니다.

## 컨텍스트 관리

Claude Code의 1M 토큰 컨텍스트는 "초보자 함정"으로 평가됨. 25만 토큰 이하로 적극적으로 관리해야 효과적이며, 이는 에이전트의 집중도와 지시 준수에 직접적인 영향을 미친다.

## 비용 비교

Codex Pro ×5 ≈ Claude ×20 수준의 사용량 상한. 동일 비용 대비 Codex가 더 많은 사용량을 제공.
