# LLM Wiki 패턴

[[andrej-karpathy]]가 제안한 LLM 기반 개인 지식 베이스 구축 패턴. [[rag]]의 대안으로, LLM이 원본 소스로부터 **영구적이고 점진적으로 축적되는 위키**를 구축·유지한다.

## 핵심 원리

- **축적**: 지식이 매 질의마다 재발견되는 것이 아니라, 한 번 컴파일되고 계속 갱신됨
- **교차 참조**: LLM이 문서 간 연결을 자동으로 생성·유지
- **모순 감지**: 새 소스가 기존 지식과 충돌하면 표시
- **복리 성장**: 소스가 추가될수록, 질의할수록 위키가 풍부해짐

## RAG와의 차이

| 항목 | [[rag]] | LLM Wiki |
|------|---------|----------|
| 상태 | 비상태 (매번 처음부터) | 상태 유지 (축적) |
| 지식 저장 | 원본 청크 검색 | 컴파일된 위키 |
| 교차 참조 | 없음 | 자동 생성 |
| 유지보수 | 불필요 | LLM이 자동 수행 |

## 3계층 아키텍처

1. **Raw sources**: 불변 원본 (사용자가 큐레이션)
2. **Wiki**: LLM이 소유·관리하는 마크다운 (사용자는 읽기만)
3. **Schema**: 위키 규칙과 워크플로 정의 파일

## 역사적 맥락

Vannevar Bush의 [[memex]](1945)와 정신적으로 연결된다 — 문서 간 연상 경로가 있는 개인 지식 저장소라는 비전. Bush가 해결하지 못한 유지보수 부담을 LLM이 해결한다.
