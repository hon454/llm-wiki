# 에이전틱 코딩 워크플로

AI 코딩 에이전트를 활용한 체계적인 소프트웨어 개발 워크플로. 단순한 "바이브 코딩"이 아닌, 계획-실행-리뷰 루프를 통해 에이전트를 관리하는 패턴이다.

## 핵심 구성 요소

### 1. Plan Mode (계획 모드)
- 충분히 스코프된 프롬프트로 작업 계획을 먼저 수립
- 구현에 앞서 아키텍처, 범위, 접근 방식을 명확히 정의

### 2. Subagent Review (서브에이전트 리뷰)
- 계획 단계에서 여러 전문 서브에이전트가 리뷰 수행
- 예시 역할: 아키텍처, 코딩 표준, UI 설계, 성능 최적화
- 각 서브에이전트에 사전 연구 문서를 참조로 전달 (예: `postgres_performance.md`, `software_architecture.md`)
- 아키텍처 리뷰어는 SOLID, DRY, KISS, YAGNI 등 원칙을 명시적으로 검증

### 3. 참조 문서 시스템
- 사전 "연구 세션"에서 도메인별 참조 문서를 생성
- 서브에이전트 프롬프트에 명시적으로 전달하여 품질 기준선을 설정

### 4. 단계별 커밋 + 코드 리뷰
- 계획의 각 단계를 별도 커밋으로 분리
- 각 커밋에 서브에이전트 기반 코드 리뷰 스킬 실행
- 수동 피드백과 조향을 병행

### 5. 프로젝트 규칙 파일
- `CLAUDE.md` (Claude Code) / `AGENTS.md` (Codex): TDD, Git 워크플로, DevEx 컨벤션, 도구 사용법 등 명시
- 약 100줄 규모가 실용적

## 바이브 코딩 vs 에이전틱 코딩

| 항목 | 바이브 코딩 | 에이전틱 코딩 |
|------|------------|--------------|
| 계획 | 없거나 즉흥적 | 명시적 plan mode |
| 리뷰 | 없음 | 서브에이전트 + 수동 |
| 품질 기준 | 암묵적 | 참조 문서로 명시 |
| 커밋 단위 | 대규모 | 단계별 분리 |
| 적합 대상 | 프로토타이핑 | 엔터프라이즈급 개발 |

## [[llm-wiki-pattern]]과의 관계

에이전틱 코딩 워크플로의 참조 문서 시스템은 [[llm-wiki-pattern]]의 축적적 지식 관리와 유사하다. 연구 세션에서 생성된 문서가 이후 개발 세션의 품질 기준으로 작용하는 구조.
